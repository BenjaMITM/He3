Introduction

He3 (Helium 3), a multi-window, floating browser for Mac OS X, Above all else.

This help will cover the few less obvious topics.

Key vs Current Window

With advent of multiple windows, it might not be obvious which is the current or key window, especially when He3 is not the default application.

The He3 key window features its title in a bold font.

When He3 is the current app, you can click on one of its windows to select a new key window.  To make He3 the current, active, application, hold down the SHIFT key when moving into a window.  You can also select a key window by using the Windows menu when He3 is active.

Note the mouse click on lower right window to make key window, but then moves outside to another, but key window is closed on user input ⌘W key press.

Windows

While He3 supports multiple windows, these are not documents, not in the typical sense of individual storage, file / URL openings, or save dialogs.  He3 treats all documents as part of a larger context stored within the user's defaults.

Each of a document's details are either read-only attributes or read-write preferences.   Document details are automatically saved whenever a URL is made known to He3 or any changes are made, such a moving a document window.  He3 does support normal URL file open and drag-and-drop operations but not save.

Document Details

  • URL
  • attributes: window size, origin, time - for audio / video URLs
  • preferences: auto-hide, float-above, opacity, translucency
  • WebView - Mac OS X built-in web browser

URL

A window is presented for each URL made known to He3 by either filesystem or web location.   All URL access is read only.  He3 can write by drag-and-drop to the Finder. Dragging a document window title bar icon creates an alias to the document url.

Attributes

Name

A window name is derived from the document URL.

Window Size

By default a window's size is remembered when recalled.

Window Origin

By default a window's origin is remembered when recalled and updated when moved.

Duration

For video and audio URLs, He3 will catalog the asset's duration.

User Agent

The global user agent preference is an inherited default which maybe altered to a custom user agent on a per view basis

WebView

Each document window is a browser window and by its contextual menu, can be alter its attributes such as mute, loop, hide controls, copy video adress, user agent, and when implemented: full screen, and download.  It can also provide access to helper applications such as Share, as well as access to open new URLs, close window, and access to document attributes, preferences as well as to access playlists and their play items.

Preferences

Auto-hide

He3 can hide and restore a document window's title when moving the pointer beyond the document window.  This preference is document specific.

Float Above

He3 will float its document windows above other apps, across all spaces. Optionally the window can be local to the current space.&ebsp; Note that windows cannot be moved across spaces.  This preference is document specific.

Opacity

He3 can vary a document window opacity value from 10 .. 100%.   This preference is document specific.

History Keep

He3 will keep the items seen in a history playlist named "History" by default; this may be altered in the Playlists panel.

The default history "keep" size is 256 items. Should you need to alter the default use the Terminal app; for example:

    1% defaults read -app He3 historyKeep
    256
    1% defaults write -app He3 historyKeep 100
    1% defaults read -app He3 historyKeep
    100

It is not advised to choose a value too large.

Location Services

He3 does not use or require location services, but some web sites, such as those featuring streaming content might. To support this, He3 features the ability to toggle location services and or disable or deny location for the app. Note that this denial can only be reset via the Security & Privacy Panel of the MacOS Settings app. He3 provides a quick link to this app via the Location Services tab of its Preferences panel:

To further simply the process of toggling location services, there's also a menu item to do so, whose state can also be viewed witin the He3 Preferences panel.

Finally, He3 can be told to remember your location preference and restore it between app executions.

Playlist Throttle

He3 will limit the size of a playlist "play" to reduce runaway sessions.

Should you need to alter the default use the Terminal app; for example:

    1% defaults read -app He3 playlistThrottle
    32
    1% defaults write -app He3 playlistThrottle 16
    1% defaults read -app He3 playlistThrottle
    16

It is not advised to choose a value too large.

User Agent

The He3 user agent string should not normally be modified. With that said however, you may need to alter it.

He3 maintains both a global preference default user agent - affecting future web views:

or per web view attribute, custom user agent setting, affecting the current view only:

This setting dialog may appear as a separate window or an attached sheet depending on the current key window if any, and whether the dialog was invoked from menu bar (default) or status app or contextual menu (custom).

Note: With the advent of tabs, a He3 window may contain several views, each with a distinct user agent.

Translucency

He3 can vary a document window opacity display when moving the pointer inside or outside a document window, never, or always.  This preference is document specific.  This video shows from top left clockwise, an auto-hide title, an always transluency, a mouse over, a mouse outside, and a never translucency window.

History

He3 remembers all documents seen in a history queue.   This information is saved and restored across sessions.

Search

He3, provides a web search interface to Bing, Google and Yahoo! web search urls.

Searches are cached within the default history playlist and preserved across sessions.

Playlists

He3 can catalog documents into playlists, groups of document URLs, which can be restored or opened by play list or its individual play items.  By default, He3 will restore all document windows left open from a prior session. Playlist items can be manually created or via drag-n-drop. Editing a playlist item is guarded by its non-zero time field.

The tableView corner menu features a button to renumber in current order playlist items. This ranking governs which item are selected for playlist. Duplicate items can be pruned by holding down SHIFT key while pressing the button.

Menus
